Revenge of Lights Out

Addendum

In creating the "Revenge of Lights Out" game, I followed a step-by-step process to ensure that the game is functional, visually appealing, and user-friendly. Here's a breakdown of my process and creative choices:

  • Understanding the Requirements: I carefully read the requirements provided, which included implementing a Lights Out game using JavaScript, ensuring the game starts with a random but solvable board configuration, and adding a footer with the last modified date.
  • HTML Structure: I began by creating the basic HTML structure for the game, including the main heading, game board container, and footer.
  • Styling with CSS: I utilized CSS to style the game board, squares, and footer. I opted for a simple grid layout for the game board and styled the squares to have a clear contrast between on and off states.
  • JavaScript Logic: I implemented the game logic in JavaScript, including toggling squares on click, checking if the game is solved, and randomizing the initial board configuration. I used event listeners to detect clicks on squares and toggled their classes accordingly.
  • Footer Last Modified Date: To dynamically display the last modified date in the footer, I used JavaScript to fetch the lastModified property of the document and update the content of the footer accordingly.
  • Testing and Debugging: I thoroughly tested the game to ensure it behaves as expected. I checked for functionality such as toggling squares, verifying the solvability of the board, and triggering the "You win!" pop-up when the game is solved. I also tested the responsiveness of the layout on different devices.
    • Overall, the process of creating the "Revenge of Lights Out" game allowed me to apply my knowledge of HTML, CSS, and JavaScript while tackling challenges related to game logic, styling, and dynamic content. It reinforced the importance of clear requirements understanding, systematic problem-solving, and creative decision-making in web development projects.
